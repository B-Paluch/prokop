O REST API

REST(Representational State Transfer) to API definiujące zestaw funkcji które programiści mogą wykorzystać w celu wysyłania żądań i otrzymywania odpowiedzi wykorzystując metody HTTP takie jak GET czy POST

API RESTowe może być używane przez każdą stronę lub aplikację niezależnie od tego w jakim języku są napisane ponieważ RESTowe żądania bazują na protokole HTTP natomiast odpowiedź jest zazwyczaj wysyłana w formacie JSON, które są wspierane przez prawie każdy język programowania.

Internet bazuje na protokole HTTP. Protokół ten pozwala na wysyłanie żądań do serwerów oraz otrzymywanie odpowiedzi które można wyświetlić w przeglądarce.

Żądanie(request) to pakiet wysyłany protokołem HTTP na który składa się:

  • Linia żądania
  • Zero, jeden lub więcej nagłówków
  • Pusta linia
  • Opcjonalne ciało żądania

Ma linię żądania składają się:Metoda Request-URI HTTP-VERSION CRLF

Metoda zawiera informację dla serwera co należy zrobić z danym żądaniem, co jest celem lub o jakie zasoby chodzi w danym żądaniu.

Request-URI to jednoznaczny identyfikator zasobów o które należy wykonać żądanie. Wykorzystywane formaty to:

  • *, która oznacza że żądanie nie odnosi się do zasobów, tylko bezpośrednio do serwera
  • Adres absolutny URI, który zawiera pełny link do zasobów o które serwer jest zapytany.
  • Ścieżka absolutna, która informuje serwer o położeniu zasobów w odniesieniu do serwera.

HTTP-VERSION to informacja o wersji protokołu HTTP wykorzystywanej przy żądaniu.

CRLF to niewidoczny ciąg dwóch znaków CR i LF które oznaczają koniec linii.

Przykładowe linie żądań:

  • OPTIONS * HTTP/1.1
  • GET http://www.w3.org/pub/WWW/TheProject.html HTTP/1.1
  • GET /pub/WWW/TheProject.html HTTP/1.1
    wtedy w treści nagłówka Host: www.w3.org

Nagłówek żądania to pole pakietu pozwalające na przesłanie dodatkowej informacji o żądaniu i kliencie do serwera. Pola nagłówka modyfikują właściwości żądania. Poniżej przykładowy nagłówek żądania:

                        User-Agent: Mozilla/4.0 (compatible; MSIE5.01; Windows NT)
                        Host: www.tutorialspoint.com
                        Accept-Language: en-us
                        Accept-Encoding: gzip, deflate
                        Connection: Keep-Alive

Ciało żądania zawiera dane przesyłane do serwera w celu przetworzenia przy żądaniu. Przykładowe ciała żądania:

                            licenseID=string&content=string&/paramsXML=string
                        
                            <?xml version="1.0" encoding="utf-8"?>
                            <string xmlns="http://clearforest.com/">string</string>

DOKUMENTACJA API

API utworzone na rzecz tego projektu zostało stworzone przy wsparciu backendu ExpressJS, z wykorzystaniem nierelacyjnej bazy danych MongoDB. Frontend został utworzony w AngularJS v11.

Backend ExpressJS jest serwowany na porcie 8080, przy czym docker mapuje ten port na port 80 urządzenia.

Frontend jest serwowany na porcie 8081.

Endpointy API

Endpointami nazywane jest połączenie metody oraz URI. Powiązujące pewne funkcje i procesy wywoływane na serwerze z danym endpointem. W projekcie utworzono poniższe endpointy:

  • GET: / | Pobiera poniższą stronę
  • GET:/api/tutorials | Pobiera informacje o wszystkich obiektach.
  • POST:/api/tutorials | Tworzy nowy obiekt.
  • GET:/api/tutorials/:id | Pobiera informacje o obiekcie o podanym id
  • PUT:/api/tutorials/:id | Aktualizuje informacje o obiekcie o podanym id
  • DELETE: /api/tutorials | Usuwa WSZYSTKIE obiekty. Tak, wszystkie.
  • DELETE: /api/tutorials/:id | Usuwa obiekt o podanym id.

CORS

CORS(Cross-Origin Resource Sharing) to mechanizm nagłówku pakietu HTTP określający jakie hosty mają dostęp do zasobów dla danego serwera. Zapytania pochodzące od obcego hosta są odrzucane.

W projekcie wykorzystywane są ustawienia dotyczące CORS dla localhost, na portach 8081 oraz 80.
